## High Priority Features

### ‚úÖ 1. Usage Pattern Presets
**Priority:** üü¢ High
**Status:** ‚úÖ Completed
**Complexity:** Medium

**Description:**
Pre-configured templates for common LLM use cases that automatically populate input/output tokens, requests per day, and other parameters.

**Use Cases:**
- **Customer Support Chatbot**: Moderate input/output with high request volume
  - Input: 400 tokens
  - Output: 200 tokens
  - Requests: 1000/day

- **Content Generation**: Low input with high output and moderate request volume
  - Input: 300 tokens
  - Output: 1000 tokens
  - Requests: 100/day

- **Data Analysis**: High input with moderate output and low request volume
  - Input: 3000 tokens
  - Output: 500 tokens
  - Requests: 50/day

- **Document Summarization**: Very high input with moderate output and low request volume
  - Input: 10,000 tokens
  - Output: 800 tokens
  - Requests: 20/day

- **Code Assistant**: Moderate input/output with moderate request volume
  - Input: 2000 tokens
  - Output: 1500 tokens
  - Requests: 150/day

- **Custom Usage**: User-defined pattern (default/current values)

**Benefits:**
- Faster onboarding for new users
- Easier to estimate costs without deep LLM knowledge
- Provides realistic examples for different use cases
- Reduces configuration errors

**Implementation Details:**
- Add preset selector UI above usage parameters
- Each preset displays icon, title, description, and key metrics
- Clicking a preset updates all relevant input fields
- Visual indication of currently selected preset
- Presets stored in data structure for easy maintenance

### ‚úÖ 2. Days Per Month Configuration
**Priority:** üü¢ High
**Status:** ‚úÖ Completed
**Complexity:** Low

**Description:**
Allow users to specify the number of active days per month (1-31) instead of assuming a full 30-day month. This enables more accurate cost projections for businesses with varying operational schedules.

**Use Cases:**
- 5-day work week businesses: 22 working days/month
- Seasonal operations: Variable days based on season
- Weekend-only services: 8-10 days/month
- 24/7 operations: 30-31 days/month

**Benefits:**
- More accurate monthly cost projections
- Better budgeting for part-time operations
- Flexibility for various business models
- Simple but high-impact improvement

**Implementation Details:**
- Add "Days Per Month" input field to Duration mode
- Default: 30 days
- Range: 1-31 days
- Update monthly calculations to use actual days instead of fixed 30
- Show active days in results summary
